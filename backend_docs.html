/join

Adds a student to our records and to the mailing list

Type: POST

Request Params:

  • firstName | The first name of the student
  • lastName | The last name of the student
  • email | The Crimson email address of the student
  • slack | "true"/"false" | Determines that the student wants a link to join the Slack team
  • shirtSize | The shirt size of the student

Sample Response:

{
    "success": true,
    "errorMessage": ""
}

/member/checkMemberForDues

Validates the member has not already paid, has already joined, and that there is an active semester for which to pay.

Type: POST

Request Params:

  • email | The Crimson email address of the student

Sample request:

{
    "email": "sarecursion@crimson.ua.edu"
}

Sample Response:

{
    "success": false,
    "errorMessage": "sarecursion@crimson.ua.edu has already paid for the current semester."
}

/member/payforsemester

Called by the payments layer, validates a payment was made and adds a payment record to the member database.

Type: POST

Request Params:

  • email | The Crimson email address of the student
  • purchaseID | The transaction id generated by Braintree
  • size | The tshirt size with which to associate the member
  • datePaid | The date of the transaction
  • paymentType | The payment type as specified by Braintree
  • last4 | The last 4 digis of the credit card used (empty for Venmo transactions)
  • cardType | The card carrier name (empty for Venmo transactions)

Sample request:

{
    "email": "jmoverflow@crimson.ua.edu",
    "purchaseID": "afdsa93",
    "size": "L",
    "datePaid": "2018-07-16",
    "paymentType": "credit_card",
    "last4": "1111",
    "cardType": "Visa"
}

Sample Response:

{
    "date": "2018-07-16",
    "hiddenCCNumber": "************1111",
    "success": true,
    "name": "Jose Overflow",
    "cardType": "Visa",
    "errorMessage": "",
    "id": "afdsa93",
    "email": "jmoverflow@crimson.ua.edu",
    "paymentType": "Credit Card"
}

/semester/add

Adds a new semester

Type: POST

PROTECTED

Request Params:

  • startDate | The start date of the semester
  • endDate | The end date of the semester
  • dueDate | The date for which to stop sending due reminders by email

Sample Request:

{
    "secretKey": "superSecret",
    "startDate": "2019-01-10",
    "endDate": "2019-05-10",
    "dueDate": "2019-03-10"
}

Sample Response:

{
    "success": true,
    "errorMessage": ""
}
